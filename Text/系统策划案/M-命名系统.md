
### 命名系统与人性标识设计

#### 系统概述
命名系统分为两大结构化子系统，分别满足功能性命名和叙事性命名的不同需求：

🔹 **程序化命名系统（Procedural Naming System）**：为技能、道具、科技、装备等游戏资源提供功能性+风格性兼备的自动化命名规则。

🔸 **人性命名演化系统（Narrative Naming Evolution System）**：承载角色人性演化、道德选择、阵营归属等叙事推进与社会互动机制。

两大系统通过统一的词根库和命名逻辑相互联动，既保证了游戏世界的命名一致性，又满足了不同层面的功能需求。

## 🔹 一、程序化命名系统（Procedural Naming System）

### 系统目标
为各类游戏资源提供功能性+风格性兼备的程序化命名规则，确保命名的一致性、可读性和文化适配性。

### 使用范围
- **技能/招式名**：武功招式、法术技能、特殊能力
- **科技/发明名**：科技研究、工艺发明、机械装置
- **装备武器名**：武器装备、防具道具、特殊物品
- **NPC功能角色名**：商人、工匠、守卫等功能性角色
- **配置表字段ID**：脚本参数、数值配比、系统标识

### 命名模板结构

| 类别 | 命名逻辑模板 | 示例 |
|------|-------------|------|
| **技能** | 【功能核心】+【表现形式】+【特效风格】 | 火突刺·烈阳式、劲风斩·旋空形、寒冰护·玄武盾 |
| **道具** | 【功效类词】+【用途或材料】 | 强筋散、银纹锁、止血布、聚气丹 |
| **科技** | 【功能目标】+【原理机制】+【阶段编号】 | 木气动链机·型二、反张力弩机·试制型 |
| **角色ID** | 【阵营简称】+【人性主类】+【岗位编号】 | 守-仁-002、革-智-001、中-勇-005 |
| **装备** | 【武器类】+【属性核心】+【品阶/出处】 | 长刀·破甲·卫都制、硬弓·劲弦·工坊制 |
| **配置表字段** | 【模块前缀】_【功能描述】 | SKL_EffectRange、EQP_DefAdd、NPC_FavorLevel |

### 程序化生成规则

#### 基础生成公式
```
[主词根] + [属性词根/用途词] + [风格修饰/版本]（可选）

示例生成过程：
1. 选择功能词根："突" (攻击类)
2. 选择属性词根："火" (火系)
3. 选择风格修饰："烈" (刚猛风格)
4. 选择表现形式："刺" (具体动作)
5. 生成结果："火突刺·烈阳式"
```

#### 高级组合规则
```
// 复合技能命名

// 科技系列命名
[研发机构] + [技术类型] + [代数版本]
示例："工部·机关术·第三代"
```

## 🔸 二、人性命名演化系统（Narrative Naming Evolution System）

### 系统目标
承载角色人性演化、道德选择、阵营归属等叙事推进与社会互动机制，通过"姓名+字号+尊号"的三层结构记录角色的人性社会化进程。

### 使用范围
- **角色命名三段式**：姓名 / 字号 / 尊号的动态演化
- **谥号系统**：去世后的最终命名评价
- **阵营改名**：阵营转换时的更名机制
- **行为加点**：道德行为对命名的影响
- **改名仪式**：重大转折时的社会认同过程
- **社会称号**：基于声望和地位的称号系统

### 命名层级结构

| 层级 | 名称 | 获取机制 | 示例 |
|------|------|----------|------|
| **基础姓名** | 姓 + 名 | 出生决定 / 家族生成 | 赵子瑜、苏景明、李慕白 |
| **字号** | 根据人性特质生成 | 人性考验 + 道德选择事件 | 慈心怀、智新进、贪财聚 |
| **尊号** | 达到权力阶段获得 | 政治地位 + 行为审判 | 智王治国、义相辅政、仁君爱民 |
| **谥号** | 死亡后评价命名 | 总结人生行为与影响 | 仁德圣祖、奸诈枭鬼、智慧文宗 |

### 事件驱动机制

#### 📍 人性节点加权系统
每次行为为不同人性特质打分，达到临界值触发命名演化事件：

```
// 行为影响权重表
救助行为: +10~30仁德, +5~15正义
欺骗行为: +8~25奸诈, -5~20诚信
勇敢行为: +15~35勇气, +5~15正义
贪污行为: +20~40贪婪, -10~30正义
学术成就: +10~25智慧, +5~10诚信
背叛行为: +25~50奸诈, -20~40诚信, -15~30正义

// 临界值触发条件
单一特质达到100点 → 触发"字号候选"事件
主要特质权重超过0.5 → 触发"字号确认"事件
特质权重发生重大变化(±0.3) → 触发"改名考虑"事件
```

#### 📍 改名仪式流程
```
事件判定 → 社会质疑 → 改名选择 → 更新命名结构 → 修改社会认同

具体流程：
1. 触发条件检测：阵营变化、重大行为背离、"名实不符"状态
2. 社会反应评估：NPC态度变化、声望影响、家族反应
3. 改名选项生成：基于新的人性特质生成候选名字
4. 玩家选择确认：选择保持原名或接受新名
5. 社会认同更新：更新NPC称呼、声望系统、历史记录
```

#### 📍 声望系统互动
```
"德名相符" → 声名远播 → NPC敬称提升 → 社会地位上升
"名实不符" → 社会质疑 → 声望削减 → 触发改名压力
"重名救赎" → 通过善行洗白 → 获得正面尊号机会
"恶名昭彰" → 社会排斥 → 被迫改名或流放
```

  （怯/胆/畏/惧）+（小/弱/退/避）
  （逃/躲/隐/缩）+（世/战/难/险）
- **示例**：
  - 怯弱退、胆小避、畏难逃、惧险者
  - Timidus（胆怯）、Ignavus（懦弱）、Fugax（逃避）
- **人性考验标识**：通过逃避责任、临阵脱逃、缺乏担当等获得

#### 阵营归属命名标识

🏛️ **传统保守派标识**
- **核心理念**：维护传统、稳定秩序、渐进改革
- **命名特征**：
  - 守护类：守正、恒德、稳固、护统
  - 传承类：承先、继志、传统、世代
  - 稳定类：安邦、定国、稳民、固本
- **称号系统**：守正者、传统卫士、稳定之柱、秩序维护者

🚀 **革新进步派标识**
- **核心理念**：推动变革、追求进步、开拓创新
- **命名特征**：
  - 变革类：革新、变通、改进、更新
  - 进步类：进取、前进、发展、提升
  - 创新类：创造、新颖、开拓、突破
- **称号系统**：革新者、进步先锋、创新领袖、变革推动者

⚖️ **中庸平衡派标识**
- **核心理念**：寻求平衡、调和矛盾、中庸之道
- **命名特征**：
  - 平衡类：平衡、和谐、调和、中庸
  - 协调类：协调、统一、融合、整合
  - 稳健类：稳健、谨慎、理性、务实
- **称号系统**：平衡者、和谐使者、调和大师、中庸智者

🔥 **激进变革派标识**
- **核心理念**：彻底变革、打破旧制、激进改革
- **命名特征**：
  - 激进类：激进、彻底、根本、深刻
  - 破立类：破旧、立新、颠覆、重建
  - 革命类：革命、起义、反抗、推翻
- **称号系统**：革命者、破旧立新者、激进改革家、颠覆者

#### 社会化阶段命名演化

👶 **家庭社会化阶段**（0-15岁）：启蒙命名
- **基础姓名**：由家族传统和文化背景决定
- **启蒙字号**：根据早期人性表现给予初步标识
- **示例**：小仁（显示仁爱倾向）、慧儿（显示智慧天赋）、勇童（显示勇敢特质）

🎓 **同伴社会化阶段**（15-25岁）：成年命名
- **正式字号**：根据青少年期的人性考验结果确定
- **阵营标识**：开始显示初步的阵营倾向
- **示例**：仁德明（仁德类+传统派）、智新进（智慧类+革新派）

💼 **职业社会化阶段**（25-40岁）：功名命名
- **职业称号**：根据职业成就和社会贡献获得
- **道德标识**：根据职业生涯中的道德表现确定
- **示例**：仁医圣手、智者导师、勇将护民、义商惠民

👑 **权力社会化阶段**（40-60岁）：尊号命名
- **权力尊号**：根据权力地位和治理能力获得
- **历史评价**：根据重大决策和历史影响确定
- **示例**：仁君爱民、智帝治国、勇王护疆、义相辅政

🏛️ **传承社会化阶段**（60岁+）：谥号命名
- **最终谥号**：根据一生的人性表现和历史贡献确定
- **传承标识**：影响后代和弟子的命名传统
- **示例**：仁德圣祖、智慧文宗、勇烈武祖、正义法祖

#### 命名试探与社会认同机制

🔍 **命名真实性试探**：
- **名实相符测试**：通过观察行为验证名字与人格的一致性
- **道德标准验证**：测试是否真正符合名字所代表的道德标准
- **阵营忠诚度探测**：验证阵营标识的真实性和忠诚度
- **社会认同评估**：评估社会对其命名的接受和认同程度

🎭 **改名重塑机制**：
- **人性转变改名**：重大人性转变后的改名仪式和社会认同
- **阵营转换更名**：阵营立场转变后的更名程序和标识更新
- **道德救赎重名**：通过道德救赎行为获得新名字的机会
- **社会流放除名**：严重违背道德标准后被社会除名的机制

🏆 **声名传播系统**：
- **美名远扬**：正面人性特质和道德行为的声名传播
- **恶名昭彰**：负面人性特质和不道德行为的恶名传播
- **名声逆转**：通过重大行为改变实现名声的逆转
- **历史定名**：历史对其最终的命名评价和定位

#### 文化传承与命名创新

📚 **传统命名文化**：
- **家族命名传统**：家族内部的命名规则和传承方式
- **地域命名特色**：不同地区的命名文化和特色
- **宗教命名影响**：宗教信仰对命名的影响和规范
- **历史命名典故**：历史人物和事件对命名的影响

🆕 **现代命名创新**：
- **时代特征融入**：将时代特征和价值观融入命名
- **跨文化命名**：不同文化背景的命名融合和创新
- **个性化命名**：根据个人特质和偏好的个性化命名
- **科技影响命名**：科技发展对命名方式和内容的影响

🔄 **命名演化规律**：
- **社会变迁影响**：社会变迁对命名趋势和偏好的影响
- **价值观演变**：价值观变化对命名标准和内容的影响
- **文化交流促进**：文化交流对命名多样性和创新的促进
- **历史周期循环**：命名文化的历史周期性变化和循环

#### 使用方式（智能命名流程）

1. **人性特质识别**：
   - 分析角色的核心人性特质（仁、智、勇、义、信等）
   - 识别主要的道德倾向和行为模式
   - 确定阵营归属和政治立场

2. **社会化阶段定位**：
   - 确定角色当前的社会化阶段
   - 分析该阶段的命名特征和要求
   - 考虑阶段转换对命名的影响

3. **文化背景融合**：
   - 选择适合的文化背景和命名风格
   - 融合家族传统和地域特色
   - 考虑宗教信仰和历史影响

4. **命名公式应用**：
   - 套用相应的命名逻辑公式
   - 结合人性特质和道德标识
   - 添加阵营标记和社会化标识

5. **社会认同验证**：
   - 评估命名的社会接受度
   - 验证名实相符的程度
   - 考虑改名和重塑的可能性

6. **动态演化机制**：
   - 建立命名随人性发展的演化机制
   - 设置重大转折点的改名触发条件
   - 记录命名变化的历史轨迹

## 🔄 三、系统联动与实施方案

### 系统分层设计

| 系统 | 设计层次 | 主要功能 | 联动接口 |
|------|----------|----------|----------|
| **程序化命名系统** | 工具逻辑、数据表构建 | 资源命名生成、配置表管理 | 技能系统、装备系统、科技树 |
| **人性命名演化系统** | 游戏机制、剧情互动 | 角色发展、社会认同、阵营判断 | 人性系统、声望系统、剧情系统 |

### 联动接口设计

#### 📊 数据接口
```json
// 程序化命名接口
{
  "generateName": {
    "input": {"type": "技能", "function": "攻击", "attribute": "火", "level": "高级"},
    "output": "烈火斩·精式"
  },
  "validateName": {
    "input": "烈火斩·精式",
    "output": {"valid": true, "category": "技能", "power_level": "高级"}
  }
}

// 人性命名接口
{
  "updateCharacterName": {
    "input": {"character_id": "001", "behavior": "救助贫民", "impact": "+30仁德"},
    "output": {"name_changed": false, "reputation_change": "+15", "next_threshold": 70}
  },
  "triggerNamingEvent": {
    "input": {"character_id": "001", "event_type": "阵营转换"},
    "output": {"new_name_options": ["革新明", "变通智"], "social_impact": "中等"}
  }
}
```

#### 🎮 游戏机制接口
```
// 与技能系统联动
技能学习 → 调用程序化命名 → 生成技能名称 → 更新技能树显示

// 与声望系统联动
行为评价 → 人性特质更新 → 命名状态检查 → 声望值调整

// 与剧情系统联动
重大选择 → 触发命名事件 → 改名仪式剧情 → 社会反应更新

// 与阵营系统联动
阵营转换 → 命名标识更新 → NPC态度变化 → 可用任务更新
```
